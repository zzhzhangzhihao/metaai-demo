## 🔄 What Is MetaAI?

MetaAI is not just another AI wrapper. It's a **methodology** for building AI systems that get smarter over time — the way a great engineer does by learning from experience.

### The Core Idea

Traditional AI systems are **linear**: Design → Build → Deploy → Maintain.

MetaAI is **recursive**: Design → Build → Deploy → Observe → Learn → Improve → Redeploy.

Every interaction generates learning material. Every success becomes a reusable skill. Every failure becomes documented wisdom.

## 🏗️ Architecture

The system is built around a **Loop Engine** with 15 capability layers:

| Layer | Name | Purpose |
|-------|------|---------|
| 1 | Coordination | Task decomposition and sub-task dispatch |
| 2 | Deep Verification | Semantic validation of all outputs |
| 3 | Meta-Cognition | Self-reflection and performance analysis |
| 4 | Skill Extraction | Automatic conversion of workflows to reusable skills |
| 5-15 | Specialized Layers | Persistence, self-modification, vision, parallel execution, observability |

## 📚 Knowledge Base: Zhiling (智令)

All learned skills, patterns, and reflections are stored in a structured knowledge base organized into 6 zones:

- **AGENT Zone**: Executable skills (92+ skills across 20+ categories)
- **ISSUE Zone**: Problems and solutions
- **METHOD Zone**: Methodologies and frameworks  
- **STRATEGY Zone**: Business and growth strategies
- **DATA Zone**: Raw data and analysis
- **WORK RECORD Zone**: Execution logs

Each entry features quality scoring, tagging, and automated governance (deduplication, archival, cross-zone migration).
